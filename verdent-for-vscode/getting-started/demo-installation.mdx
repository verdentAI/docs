# Installing the Example Development Tool

This demo page showcases the **Installation Guide** template pattern. This template is designed for comprehensive installation documentation covering multiple platforms, system requirements, and troubleshooting.

<Note>
This is a **demo page** showing template structure, not actual Verdent documentation. Notice the multi-platform Tabs, system requirements table, and focused installation-only scope characteristic of installation guides.
</Note>

### Overview

The Example Development Tool is a command-line interface and editor extension that enhances your development workflow. This guide walks you through installation on all supported platforms.

**Installation time**: 5-10 minutes

### System Requirements

Before installing, verify your system meets these requirements:

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| **Operating System** | macOS 11+, Windows 10+, Ubuntu 20.04+ | macOS 13+, Windows 11, Ubuntu 22.04+ |
| **Processor** | Intel Core i3 or equivalent | Intel Core i5 or Apple M1+ |
| **Memory (RAM)** | 4 GB | 8 GB or more |
| **Disk Space** | 500 MB free | 2 GB free |
| **Node.js** | 18.x or higher | 20.x LTS |
| **Package Manager** | npm 9+, yarn 1.22+, or pnpm 8+ | pnpm 8+ (fastest) |
| **Internet** | Required for installation | Broadband recommended |

<Warning>
**Important Prerequisites:**

- You must have Node.js installed before proceeding
- Administrator/sudo access required for system-wide installation
- Ensure your firewall allows npm registry access (registry.npmjs.org)
</Warning>

## Installation Methods

Choose the installation method that best fits your workflow:

#### Method 1: Package Manager (Recommended)

The fastest way to install across all platforms using your system's package manager.

<Tabs>
  <Tab title="macOS">
    **Using Homebrew:**

    ```bash
    # Install via Homebrew
    brew install example-dev-tool

    # Verify installation
    example-dev-tool --version
    ```

    **Using npm:**

    ```bash
    # Install globally
    npm install -g example-dev-tool

    # Verify installation
    example-dev-tool --version
    ```

    **Post-installation:**

    ```bash
    # Initialize in your project
    cd your-project
    example-dev-tool init
    ```
  </Tab>

  <Tab title="Windows">
    **Using Winget:**

    ```powershell
    # Install via Windows Package Manager
    winget install ExampleDevTool

    # Verify installation
    example-dev-tool --version
    ```

    **Using npm:**

    ```powershell
    # Install globally (run as Administrator)
    npm install -g example-dev-tool

    # Verify installation
    example-dev-tool --version
    ```

    **Post-installation:**

    ```powershell
    # Initialize in your project
    cd your-project
    example-dev-tool init
    ```

    <Tip>
    **Windows Users**: If you encounter permission errors, run your terminal as Administrator or use `npm install -g example-dev-tool --force`.
    </Tip>
  </Tab>

  <Tab title="Linux">
    **Using apt (Debian/Ubuntu):**

    ```bash
    # Add repository
    curl -fsSL https://example.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/example-archive-keyring.gpg

    echo "deb [signed-by=/usr/share/keyrings/example-archive-keyring.gpg] https://repo.example.com stable main" | sudo tee /etc/apt/sources.list.d/example.list

    # Install
    sudo apt update
    sudo apt install example-dev-tool

    # Verify installation
    example-dev-tool --version
    ```

    **Using npm:**

    ```bash
    # Install globally
    sudo npm install -g example-dev-tool

    # Verify installation
    example-dev-tool --version
    ```

    **Post-installation:**

    ```bash
    # Initialize in your project
    cd your-project
    example-dev-tool init
    ```

    <Tip>
    **Linux Users**: If using npm without sudo, configure npm to use a different directory: `npm config set prefix ~/.npm-global` and add `~/.npm-global/bin` to your PATH.
    </Tip>
  </Tab>
</Tabs>

#### Method 2: Direct Download

Download pre-built binaries for offline installation or when package managers aren't available.

1. **Download the installer**:
   - Visit [https://example.com/downloads](https://example.com/downloads)
   - Select your platform (macOS, Windows, or Linux)
   - Choose your architecture (x64, arm64, or arm)

2. **Run the installer**:
   - **macOS**: Open the `.dmg` file and drag to Applications
   - **Windows**: Run the `.exe` installer and follow the wizard
   - **Linux**: Extract the `.tar.gz` and run `./install.sh`

3. **Verify installation**:
   ```bash
   example-dev-tool --version
   ```

#### Method 3: Build from Source

For advanced users who need the latest development version or want to contribute.

```bash
# Clone the repository
git clone https://github.com/example/dev-tool.git
cd dev-tool

# Install dependencies
npm install

# Build the project
npm run build

# Link globally
npm link

# Verify installation
example-dev-tool --version
```

## Verifying Your Installation

After installation, verify everything is working correctly:

```bash
# Check version

example-dev-tool doctor
```

Install editor extensions for enhanced IDE support:


2. Navigate to Plugins
3. Search for "Example Dev Tool"
4. Click Install and restart IDE

## Configuration

Create a configuration file to customize behavior:

```bash
# Generate config file
example-dev-tool config init
```

This creates `example-config.json`:

```json
{
  "version": "1.0",
  "autoUpdate": true,
  "telemetry": false,
  "editor": {
    "formatOnSave": true,
    "tabSize": 2
  },
  "build": {
    "target": "es2020",
    "minify": true
  }
}
```

<Tip>
**Configuration Tip**: Set `telemetry: false` if you prefer not to send anonymous usage data. All telemetry is opt-in by default.
</Tip>

## Troubleshooting

#### Common Installation Issues

**Issue: "Command not found" after installation**

**Solution:**

```bash
# macOS/Linux: Add to PATH in ~/.bashrc or ~/.zshrc
export PATH="$PATH:$HOME/.npm-global/bin"

# Windows: Add to PATH via System Environment Variables
# 1. Search "Environment Variables" in Start menu
# 2. Edit PATH variable
# 3. Add: C:\Users\YourName\AppData\Roaming\npm
```

Then restart your terminal.

**Issue: Permission denied errors on macOS/Linux**

**Solution:**

```bash
# Option 1: Use sudo (quick but not recommended)
sudo npm install -g example-dev-tool

# Option 2: Fix npm permissions (recommended)
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# Then install without sudo
npm install -g example-dev-tool
```

**Issue: Installation hangs or times out**

**Solution:**

```bash
# Clear npm cache
npm cache clean --force

# Try with increased timeout
npm install -g example-dev-tool --fetch-timeout=60000

# Or use a different registry
npm install -g example-dev-tool --registry=https://registry.npm.taobao.org
```

**Issue: Version mismatch or outdated installation**

**Solution:**

```bash
# Check current version
example-dev-tool --version

# Update to latest version
npm update -g example-dev-tool

# Or reinstall
npm uninstall -g example-dev-tool
npm install -g example-dev-tool
```

**Issue: Windows Defender or antivirus blocking installation**

**Solution:**

1. Temporarily disable Windows Defender during installation
2. Add npm cache directory to exclusion list: `%AppData%\npm-cache`
3. Re-enable Windows Defender after installation

## Updating

Keep your installation up to date to receive the latest features and bug fixes:

```bash
# Check for updates
example-dev-tool update check

# Update to latest version
example-dev-tool update

# Or use package manager
npm update -g example-dev-tool  # npm
brew upgrade example-dev-tool   # Homebrew
```

**Auto-update**: Enable automatic updates in your config:

```json
{
  "autoUpdate": true,
  "updateChannel": "stable"  // or "beta"
}
```

## Uninstallation

To completely remove the Example Dev Tool:

<Tabs>

  </Tab>

    ```bash
    # Using apt
    sudo apt remove example-dev-tool

    sudo npm uninstall -g example-dev-tool

    ```
  </Tab>
