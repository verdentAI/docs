# Verdent Docs

Welcome to the official documentation for Verdent, an advanced coding assistant that helps developers manage complex software projects using multiple AI agents.

View the live documentation at [docs.verdent.ai](https://docs.verdent.ai)

## About Verdent

**Your code, your vision. Verdent makes it happen.**

Verdent is available in two versions:
- **Verdent Deck**: MacOS Desktop App
- **Verdent for VS Code**: VS Code Extension

Key features include running multiple AI agents in parallel, code review capabilities, DiffLens for explaining code changes with context, and full control over AI subagents.

Learn more at [verdent.ai](https://verdent.ai?ots=verdentdoc)

## Contributing

We welcome contributions to improve the Verdent documentation! Here's how you can contribute:

### Fork and Clone

1. Fork this repository to your GitHub account
2. Clone your fork locally:
   ```
   git clone https://github.com/YOUR_USERNAME/docs.git
   cd docs
   ```

### Make Your Changes

1. Create a new branch for your changes:
   ```
   git checkout -b your-feature-branch
   ```

2. Make your documentation changes in the MDX files

3. Test your changes locally by running:
   ```
   mint dev
   ```

4. Preview your changes at `http://localhost:3000` to ensure everything looks correct

### Submit a Pull Request

1. Commit your changes with a clear, descriptive message:
   ```
   git add .
   git commit -m "Description of your changes"
   ```

2. Push your branch to your fork:
   ```
   git push origin your-feature-branch
   ```

3. Open a pull request from your fork to the main repository

4. In your pull request description:
   - Explain what changes you made and why
   - Include screenshots if you modified the UI or layout
   - Reference any related issues

### Guidelines

- Follow the existing documentation style and formatting
- Test all code examples before submitting
- Keep changes focused and atomic
- Write clear commit messages
